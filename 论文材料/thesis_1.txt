【阶段一：基础设施冷启动（Infrastructure Cold Start）论文素材】

1. 学术背景/动机
本阶段的研究动机在于构建一个可复现实验环境，使“基于大语言模型的代码安全分析”从概念走向可验证的工程闭环。对于安全研究而言，若缺少稳定的输入/输出通路与可观测性，将难以开展后续的模型对比、提示词实验与安全性评估。因此，阶段一重点解决两个问题：（1）以契约化 API 形式定义分析任务的输入/输出；（2）在前端可交互界面中完成最小可用闭环，为后续实验迭代提供载体。

2. 核心方案说明
阶段一采用“前后端分离 + HTTP API 契约”的最小系统架构：
（1）后端基于 FastAPI 提供 HTTP 服务，并使用 Pydantic V2 定义请求/响应模型，形成严格的 Schema 约束；
（2）提供健康检查端点用于可用性探测；
（3）跨域访问通过 CORS 中间件实现，为浏览器端前端调用后端 API 打通路径；
（4）配置管理以 .env 为中心，避免将第三方模型密钥硬编码进入代码库（可视为“最小秘密管理”实践）。

从系统工程角度，该阶段的核心贡献是将“分析能力”抽象为稳定的接口契约，使后续阶段的 LLM 接入、流式输出与安全加固均可围绕契约演进。

3. 论文加分项（Academic Highlights）
（1）契约优先（Contract-first）思路：用 Pydantic schema 作为形式化约束，可在论文中解释为“输入域/输出域的显式建模”，有助于提高系统鲁棒性与可测试性。
（2）配置分离与密钥管理的最小实现：通过 `.env` 隔离敏感信息，体现安全工程的基本原则（最小暴露面）。
（3）可复现闭环作为实验平台：前端界面 + 后端 API 的组合形成“人机交互实验台”，便于开展用户体验与延迟感知研究。

4. 实验/验证数据
（1）健康检查：`GET /health` 返回 HTTP 200，body 包含 `{"status":"ok","version":"1.0.0"}`。
（2）单元测试：在阶段推进过程中已建立并运行最小测试集（累计 3 条测试通过，见 `backend/tests/`），为后续阶段的回归测试提供基础。
（3）端口可达性：后端默认运行在 `http://localhost:8000`，前端开发服务器在 `http://localhost:5173`。
